РУКОВОДСТВО ПО СОЗДАНИЮ КВЕСТОВ ДЛЯ ИГРЫ "ДОМОВЯТА"
Инструкция для ИИ-агентов по созданию Этапа 3

ЦЕЛЬ ЭТАПА 3

Этап 3 превращает сюжетные квесты из этапов 1-2 в план игровых заданий.

На этом этапе ИИ должен:
- прочитать контекст квеста: `title_quest`, `description`, `congratulation`, `Character`;
- творчески выбрать подходящие игровые механики;
- для каждого задания указать актуальный ID шаблона этапа 4;
- рядом указать русское название шаблона, чтобы план был понятен человеку;
- рядом указать машинный `Task type`, чтобы следующий Python-этап мог разобрать план.

На этом этапе ИИ НЕ должен:
- заполнять JSON `Tasks`;
- выбирать конкретные `garbage_classname`, `flower_classname`, `collection_classname`;
- придумывать игровые classname, коллекции, мусор, цветы или локации;
- делать CSV.

Конкретные игровые данные подбираются позже на этапе 4 из:
- `data/quest_ready_index.json`
- `data/quest_ready_drops.index.json`

ГЛОССАРИЙ КЛЮЧЕВЫХ ТЕРМИНОВ

Термин | Описание
-------|----------
Task template ID | Стабильный ID шаблона этапа 4, например `TT-020`
Task template name | Русское название шаблона из актуальной инструкции этапа 4
Task type | Машинное имя типа задания для парсера и следующего этапа
HOG | Hidden Object Game, поиск скрытых предметов
GR | Game Resource, временный ресурс квестового пакета
Collection | Реальный элемент коллекции из quest-ready индексов
ASK | Общая просьба предмета у друзей
PER | Личная просьба предмета у друзей
FA | Ресурс/действие на площади у друга
Mystery | Задание-загадка, обычно с `is_hide`
is_silhouette | Задание с силуэтом предмета
in_guest | Действие выполняется в гостях у друга

ОБЯЗАТЕЛЬНЫЙ ФОРМАТ ВЫХОДА ДЛЯ КАЖДОГО КВЕСТА

Каждый квест должен содержать три синхронизированные строки:

```text
Task template ID: TT-020 / TT-014 / TT-033
Task template name: Уборка конкретного мусора в гостях / GR с конкретного мусора в гостях / Передача предмета
Task type: garbage classname in_guest / get_asset GR in_guest garbage classname / action give
```

Правила:
- В строках должно быть одинаковое количество элементов.
- Порядок элементов должен совпадать с `№ task`.
- Разделитель между заданиями строго ` / `.
- `Task template ID` брать только из каталога ниже.
- `Task template name` писать по-русски точно как в каталоге.
- `Task type` писать точно как в каталоге, без произвольных синонимов.
- Если шаблон помечен `not_ready`, не выбирать его.

КРИТИЧЕСКИ ВАЖНЫЕ ПАТТЕРНЫ

1. Не повторять один и тот же шаблон без причины

В одном квесте лучше использовать разные `Task template ID`.

Допустимые исключения:
- mystery-квест, где все 3 задания являются загадками;
- silhouette-квест, где 2-3 задания являются силуэтами;
- сюжетно оправданная пара `TT-018` + `TT-019`, если нужно собрать цветы дома и в гостях.

2. Mystery всегда группой

Если выбран mystery-формат, в квесте должно быть 3 mystery-задания.

Допустимые mystery-шаблоны:
- `TT-022` Загадка на цветок в гостях
- `TT-023` Загадка на цветок дома
- `TT-024` Загадка на мусор в гостях
- `TT-025` Загадка на мусор дома
- `TT-026` Загадка на коллекцию
- `TT-027` Загадка задом наперед на коллекцию

Mystery лучше размещать ближе к финалу пакета или в отдельном загадочном сюжетном блоке.

3. Silhouette лучше группой

Silhouette-задания лучше давать группой 2-3 задания в одном квесте.

Допустимые silhouette-шаблоны:
- `TT-028` Силуэт элемент коллекции
- `TT-029` Силуэт мусора в гостях
- `TT-030` Силуэт мусора дома
- `TT-031` Силуэт цветка в гостях
- `TT-032` Силуэт цветка дома

4. Подготовка к крафту или передаче предмета

Если квест заканчивается созданием или передачей важного предмета, первые два задания должны готовить ресурсы:

```text
get_asset GR/PER/ASK/Collection -> get_asset GR/PER/ASK/Collection -> TT-002 или TT-033
```

Выбирать:
- `TT-002 Крафт`, если игрок должен создать/собрать предмет;
- `TT-033 Передача предмета`, если игрок должен отдать/применить уже подготовленный предмет.

5. Гостевые задания нужны часто

Для социальных квестов чаще выбирать гостевые варианты:
- `TT-014` GR с конкретного мусора в гостях
- `TT-016` GR с цветов в гостях
- `TT-019` Сбор цветов в гостях
- `TT-020` Уборка конкретного мусора в гостях
- `TT-024` Загадка на мусор в гостях
- `TT-029` Силуэт мусора в гостях
- `TT-031` Силуэт цветка в гостях

6. Не использовать `TT-035`

`TT-035 Следы с ресурсами` пока помечен `not_ready`, потому что в актуальной таблице нет заполненной структуры task object.

КРИТИЧЕСКИЕ ОШИБКИ ЭТАПА 3

Нельзя:
- выдавать только `Task type` без `Task template ID`;
- писать русское название шаблона не из каталога;
- писать машинный `Task type` не из каталога;
- выбирать `TT-035`;
- заполнять JSON `Tasks` на этапе 3;
- придумывать реальные игровые classname мусора, цветов, коллекций или локаций;
- использовать `mystery` одиночным заданием;
- использовать `is_silhouette` одиночным заданием без сюжетной причины;
- ставить крафт/передачу без подготовки ресурсами;
- игнорировать контекст `description` и `congratulation`;
- оставлять без блока `Обоснование`.
