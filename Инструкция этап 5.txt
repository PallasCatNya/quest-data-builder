# ЭТАП 5: ПРЕОБРАЗОВАНИЕ В ФОРМАТ CSV

Инструкция по созданию CSV для квестов:

ОБЯЗАТЕЛЬНОЕ УСЛОВИЕ ПЕРЕД CSV

Этап 5 можно запускать только после успешной проверки этапа 4:

```bash
python src/validate_task_objects.py output/filled_tasks.json
```

Если в `output/filled_tasks.validation.json` есть `errors > 0`, CSV не создавать. Нужно вернуться к этапу 4 и исправить task object.

Этап 5 не должен творчески переписывать данные. Он только разворачивает уже готовые данные этапа 4 в CSV:
- верхний блок квеста из полей `classname_quests`, `title_quest`, `description`, `congratulation`, `Character/character`, `helper`, `extra.sequence_icon`;
- блоки задач из уже готовых `task_object`.

Нельзя на этапе 5:
- менять `title`;
- менять `hint`;
- менять `classname`, `param`, `icon`;
- менять location title после фраз `Место поиска:` или `Он находится`;
- перенумеровывать generated-сущности;
- заменять мусор, цветок, коллекцию или выбранного кандидата.

Если в task object видна смысловая ошибка, этап 5 должен остановиться и вернуть задачу на этап 4.

СТРУКТУРА CSV

Для каждого квеста нужен отдельный quest-блок, а под ним его задачи:

```text
Квест 1
  Таск 1
  Таск 2
  Таск 3

Квест 2
  Таск 1
  Таск 2
  Таск 3
```

Важно: номера в заголовках задач CSV локальные внутри квеста. Даже если во входном `filled_tasks.json` `task_number` идет глобально как `4`, `5`, `6`, в блоке второго квеста CSV должен писать `Таск 1`, `Таск 2`, `Таск 3`, а поля остаются `tasks.0`, `tasks.1`, `tasks.2`.

Quest-блок должен идти перед task-блоками:

```csv
;Квест 1;;;;;;
sl;string;string;string;string;string;string;string
;input;output;title;description;congratulation;helper;extra.sequence_icon
;/quest/generated/Labor_2025/story_1/Labor_2025_Story_1.proto.js;/quest/generated/Labor_2025/story_1/Labor_2025_Story_1.proto.js;Потерянный мангал;...;...;Labor_2025_Character_1;MagazinePage_Labor_2025_QuestIcon_1
```

Поля quest-блока:
- `input` и `output` берутся из `proto_path`, если он есть, иначе строятся из `classname_quests/classname` и `quest_number`;
- `title` берется из `title_quest`;
- `description` берется из `description`;
- `congratulation` берется из `congratulation`;
- `helper` берется из `helper`, `quest_helper`, `character_classname`, `extra.helper` или из первого `task_object.go_to_location[].classname`/`icon`, похожего на `Character`;
- `extra.sequence_icon` берется из `extra.sequence_icon`, `sequence_icon`, `quest_sequence_icon`; если поля нет, строится по classname квеста как `MagazinePage_[base]_QuestIcon_1`.
